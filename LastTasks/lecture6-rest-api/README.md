## Задание 6

Так как в задании к лекции 5 не были указаны названия сущностей.
Мной были созданы Visitor (вместо User), Restaurant, Review

Реализовать RESTful API для системы оценки ресторанов
(предыдущее задание), используя Spring Boot. API должно
предоставлять CRUD-операции для работы с посетителями,
ресторанами и отзывами. Все данные должны передаваться через
DTO (Data Transfer Object), а не через сущности напрямую.  

Требования:
1. Создание DTO  
Для каждой сущности (User, Restaurant, Review) создать отдельные классы DTO:
- RequestDTO – для входящих данных(создание/обновление)
- ResponseDTO – для исходящих данных (ответ API)

<i>Использовать неизменяемые DTO (record или Lombok @Value).</i>

2. REST-контроллеры
Реализовать контроллеры для:
- Посетителей (/api/users)
- Ресторанов (/api/restaurants)
- Отзывов (/api/reviews)

<i> Поддерживать стандартные HTTP-методы (GET, POST, PUT, DELETE).</i>

3. Сервисный слой  
Обновить сервисы для работы с DTO.
4. Подключить Swagger  
Описать все эндпоинты, модели запросов/ответов.

### Дополнительное задание***
#### Требования:
1. Маппинг DTO и сущностей   
Использовать MapStruct или ModelMapper для преобразования: RequestDTO → Entity (при сохранении) Entity → ResponseDTO (при возврате данных)  
Создать отдельные интерфейсы/классы мапперов.
2. Валидация  
Настроить валидацию для входных данных с помощью аннотации @Valid. Как настраивать ее и DTO посмотреть самим.
